Version 2.2.1.0 (2024-10-26)

ANSI/IES TM-30 color rendition is fully implemented
JMOSpectrumPath returns the path to the spectrum library, for easy "addpath" to subdirectories TODO:Documentation
ScaleSpectrum now accepts an optional argument "factor" for simple scalar multiplication.
PlotSpectrum now sets xlabel to "Wavelength (nm)", which can be overriden after plotting if desired
AddWeightedSpectra: Now additional lam values are inserted just outside the range of each overlapping spectrum. This effectively adds a sudden drop to zero outside each spectrum before adding. Now, the resulting spectrum has the same effective discontinuities as the individual spectra in case they are not zero at the limits of the wavelength range.

Version 2.2.0 (2024-08-26)

Library is now based on official CIE data from https://cie.co.at/data-tables.
  • The CIEData.m class contains all 35 CIE datasets and makes them accessible to this library
  • All routines that use CIE spectral data have been updated to use CIEData.m
  • All previous .mat files with CIE spectral data from other sources have been moved to ./compost
PlanckLocus now has xy information for Judd Lines: x, y, dx, dy
updated WriteLighttoolsSpectrumFile.m with datetime
CIE_S_026_Data adds access to CIE S026:E2018 human eye sensitivities (L/M/S cones, rods, melanopic sensitive ganglion cells (ipRGCs)
Changed sRGB_to_XYZ to return matching size (bugfix)
Changed XYZ_to_sRGB to return matching size (interface change)
Changed XYZ_to_sRGB to accept single X as array or struct with .X, .Y, .Z. This allows to directly use output of CIE_1931XYZ

Version 2.1.3.1 (2023-01-25)

Fixed bug in CIE_upvp.m

Version 2.1.3 (2023-01-24)

PlotSpectrum.m, convenience function to avoid plot(s.lam, s.val), and errors like plot(good.lam, bad.val)
PlotCIExyBorder: Added "Opacity" parameter to create semi-transparent color fill.
PlotCIEupvpBorder: Added "Opacity" parameter to create semi-transparent color fill.
CIE1964_Data.m adds access to CIE1964 10° color matching functions

Version 2.1.2 (2022-05-30)

Added ScaleSpectrum.m to ensure the (possibly weighted) integral or the peak of a spectrum matches a desired value.

Version 2.1.1 (2021-09-20)

Added DivideSpectra.m (complementing MultiplySpectra) to divide one spectrum by another.
